最佳实践

让 Buddy 更好用的做法

先把问题说清楚，再让两个 AI 工具轮流工作。任务越清楚，审查越有效，最后越容易收敛。

任务说明模板

目标：一句话说明要交付什么。

资料：放上 plan.md 路径、会话 ID、截图或关键文件路径。

边界：说明哪些文件、行为或方案不要碰。

完成标准：写清楚测试、页面效果、审查结论或其他可验证结果。

一些非常不错的用法

Buddy 的价值不是让两个 AI 同时说话，而是让一个负责推进，一个负责检查，直到问题真正收口。

方案定好后，结对开发

先和人或其他工具把需求梳理清楚，形成 plan.md。启动 Buddy 时，把 plan.md 路径写进任务说明。

一个 AI 按方案实现，另一个 AI 持续检查方向、遗漏和风险。

先写完，再协作审查

单个 AI 已经完成一轮编码后，把原会话 ID 接入 Buddy，让两边沿着同一段上下文继续。

适合修复遗漏、补测试、清理边界问题。

拿不准时，先充分讨论

如果你对方案、取舍或判断拿不准，直接把问题写进任务说明，要求双方先讨论。

输出重点应是清晰结论、理由和下一步建议，而不是马上改代码。

自定义提示词，给 Agent 立规矩

在「设置 - 提示词」里写自定义提示词，它会追加到每个 Agent 的系统提示词末尾。可以调高或调低两个 Agent 的对抗强度，也可以明确禁止某些行为。

想让双方更激烈地辩论就调高强度，想更克制地推进就调低。也能用来固定输出格式、保护关键文件。
